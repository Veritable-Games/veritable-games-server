## The Critical Distinction

### What the Fix Intended:
- "Filter deleted journals from the UI" (hide them from view)

### What the Fix Actually Did:
- "Filter deleted journals from the database query" (exclude them from data entirely)

### Why This Breaks Everything:

The UI has **two separate concerns**:

1. **Data Loading** (Server): Load ALL journals (including deleted ones)
   - Needed for: Undo/redo, recovery, complete state management

2. **Display Filtering** (Client): Choose what to show
   - Can filter by: Active only, archived only, include deleted, etc.
   - The store's `getJournalsByCategory()` already handles this with priority sorting

**The fix broke concern #1**, which made concern #2 impossible to implement.
